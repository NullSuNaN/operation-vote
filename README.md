# Operation Voting

have real-time votes on keyboard/mouse actions.

**If you are here to find a way to connect to a server with preset protocol, use
[the Preset Client](http://nullsunan.page.gd/)**

## Note

You can make your own servers and clients with the `core/` stuff.

The `client-window`, `client-browser`, `server` are all just an implementation to process generic voting.

If the server port is not open for WLAN, you need to use `0.0.0.0` or your WLAN IP as the host.
If it still isn't working, then it is probably caused by your system security system.

For the detail of the server configuration, see [config.schema.json](server/config.schema.json).

## Authentication

There is an anonymous user by default, which can be configured to be disabled(vote does not count) by the server.

You can change the multiplier of each account while keeping the server on.

Every user have a username and a password, they both have no limitations on the characters and is *CASE SENSITIVE*.

You can only login with your client, and only the server host can register.

The protocol allows the client to change account mid-connection, but currently cannot change back to Anonymous and it is not implemented to the clients.

The account with the name `Anonymous` is separate from the default Anonymous user.

The users on the server-side is stored in `users.db`.

## Project Structure

+ [core/](core/): The core protocol, responsible for transforming operations and count votes.
+ Presets: [server](server/), [client-window](client-window/) and [client-browser](client-browser/) is the server and clients of the preset, they control the [instructions protocol](shared/InstructionsProtocol.cs). 
+ [server/](server/): A server interface preset, can press keys or send console messages, responsible for actions, user database and user management.
+ [client-window/](client-window/): A executable client preset, responsible for key detecting and afk processing.
+ [client-browser/](client-browser/): A webpage client preset, responsible for key detecting and afk processing.
+ [shared/](shared/): Shared resources of [server](server/), [client-window](client-window/) and [client-browser](client-browser/).
+ [release/](release/): Would be generated by [`createRelease.sh`](createRelease.sh), holds the built releases.
+ [page/](page/): For hosting the webpage client via github pages, don't use that, use [http://nullsunan.page.gd/](http://nullsunan.page.gd/) instead.

## Examples

+ Commands
```bash
server # uses config.json
server gd.json # uses gd.json
server -- gd.json # uses gd.json
server --manager gd.json # opens the user manager with gd.json
server --help # show help
dotnet serve -d:dist/wwwroot -p8080 -a 0.0.0.0 # serve the browser client, require dotnet-serve tool
client-window localhost:11167 # connect to localhost:9055 with raw TCP
client-window localhost:11169 ws # connect to localhost:9055 with WS
client-window localhost:11169 wss NullSuNaN RubRubPower123 # connect to localhost:9055 with WS and attempt to login as NullSuNaN
                                                           # or continue as Anonymous if the login failed
```
+ Server Configuration
```json
{
  "$schema": "config.schema.json",
  "Network": {
    "TcpHost": "0.0.0.0",
    "TcpPort": 11167,
    "WsUriPrefix": "http://+:11169/"
  },
  "Profiles": [
    {
      "Name": "jump",
      "Keys": ["w", "W", " ", "ArrowUp", "MouseLeft"],
      "AfkLimit": "00:00:07",
      "VoteResults": [
        {
          "type": "PressKey",
          "key": " ",
          "requireSupportRate": 0.50
        },
        {
          "type": "Output",
          "id": "antiJump",
          "fd": 2,
          "requireSupportRate": -0.2
        }
      ]
    },
  ],
  "Alert": false,
  "Logging": {
    "LogLevel": "debug",

  }
}
```
